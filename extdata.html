Create a delayed assignment for a dataset

Usage

extdata(..., assign.env = parent.frame())


extdata_(..., .dots, assign.env = parent.frame())

Arguments

...
Expressions for which delayed assignments are created.
assign.env
The environment in which to create the delayed assignments.
.dots
Expressions as lazy objects.

Description

This function creates one or more delayed assignments. It is focused on data stored in the extdata directory, and therefore especially useful for package development. The main advantages over R's internal data loading mechanism are faster package build times; the delayed assignment makes sure that the data is loaded only if necessary.

Details

The "dots" argument can be named or unnamed; for unnamed arguments, names will be assigned by all_dots. For each argument, a delayed assignment with the specified name is created via delayedAssign. The delayed assignment will evaluate the argument; if the argument is a single name, the corresponding .rds data file from the extdata directory will be loaded.

The function extdata_ is a standard-evaluation version of the above. See the "lazyeval" vignette for details.

Examples

## Not run: 
# # Creates a delayed assignment "fortytwo" which loads the file
# # "extdata/fortytwo.rds"
# extdata(fortytwo = readRDS(system.file("extdata", "fortytwo.rds",
#         package = "mypackage")))
# # A shorter version of the above
# extdata(fortytwo)
# ## End(Not run)
